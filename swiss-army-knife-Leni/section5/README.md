Security and robustness checklist

For future automation projects I will:

1. Make sure to use Python's virtual environment (.venv) to isolate projects.
2. Not commit secrets, for example passwords, to GitHub
3. I will not store sensitive inofrmation in the .env file
4. I will make sure .env is in the .gitignore file
5. I will log things instead of just printing them
6. Not use shell = True with the subprocess.run() so that users cannot input
their own commands
7. Test scripts and double check everything before pushing them to GitHub
